# Project-Scoped Rules

## 1. 디자인 철학 및 UI/UX 품질 유지 (Chzzk Theme)
- **치지직 테마 중심 디자인**: 최근 개편된 치지직(Chzzk) 스타일의 테마(Chzzk Neon Green `#00ffa3`, Chzzk Green `#00b872`, 다크 모드 `#070914`, 데스크톱 라이브/채팅 레이아웃 등)가 프로젝트의 핵심 디자인 아이덴티티입니다.
- **UI/UX 일관성**: 새로운 UI 컴포넌트나 수정 사항 반영 시, 기존 치지직 스타일의 네온/다크 무드, 둥근 모서리, 글래스모피즘, 포인트 컬러 활용 원칙을 준수하여 시각적 완성도와 일관성을 유지해야 합니다.

## 2. 작업 완료 후 Chrome Dev Tool 검증 및 증적 기록 (Verification & Evidence)
- **실제 브라우저 검증**: 프론트엔드/UI 관련 작업이 끝난 후에는 반드시 Chrome Dev Tool (또는 브라우저 서브에이전트, `take_screenshot` 툴)을 통해 변경 사항이 올바르게 렌더링되는지 직접 확인합니다.
- **스크린샷 및 영상 증적**: 변경 사항이 반영된 화면의 스크린샷이나 영상을 촬영하여 artifacts 디렉토리에 저장하고, 작업 완료 후 작성하는 `walkthrough.md`에 반드시 이미지/영상 형태로 임베드하여 증적을 남겨야 합니다.

## 3. 작업 완료 및 커밋 정책 (Commit Policy)
- **매 턴의 작업 종료 시 필수 커밋**: 매 턴의 작업을 마칠 때마다(AI 에이전트의 턴이 종료되고 사용자 피드백을 받기 전) 작업 내용을 요약한 Git 커밋을 반드시 생성하여 저장소의 안정성과 쉬운 롤백 가능성을 유지합니다.
- **커밋 메시지**: 변경 내용의 성격에 맞춰 적절한 태그(feat, style, fix, refactor 등)와 함께 작업 내용을 요약하여 작성합니다.
